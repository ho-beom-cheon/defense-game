# Stage 1 전장 아트 일관성 설계 v1

## 1. 결정 요약

Stage 1 전장의 시각 기준은 배경이 아니라 현재 영웅·몬스터 캐릭터 자산으로 삼는다.

- 목표 스타일: `고해상도 픽셀풍 판타지 일러스트`
- 핵심 인상: 어두운 봉문 숲, 황동과 석재로 만든 수호 제단, 청색 룬광, 보랏빛 균열
- 전장 정보는 유지하되 레인, 배치 슬롯, 수정체 영역을 UI 사각형처럼 그리지 않는다.
- 신규 자산은 Stage 1에 필요한 배경·레인·수정체·균열·접지 그림자로 제한한다.
- 캐릭터, 전투 좌표, 판정, 밸런스, 저장 데이터는 변경하지 않는다.

현재 UI 콘셉트인 `봉문 전술 기록서`는 유지한다. 이번 설계는 기록서 프레임 안쪽의 전장을 캐릭터와 같은 세계로 보이게 만드는 작업이다.

## 2. 현재 문제와 원인

| 구분 | 현재 상태 | 발생하는 문제 |
| --- | --- | --- |
| 캐릭터 | 419~1536px급 고해상도 픽셀풍 이미지 | 재질, 명암, 외곽선 정보가 풍부함 |
| 배경 | `bg_goblin_forest_lanes.png`, 1280×720 | 캐릭터보다 묘사 밀도가 낮고 화면에 비균등 확대됨 |
| 레인 | 반투명 직사각형과 직선 | 게임 월드가 아니라 디버그 가이드처럼 보임 |
| 수정체 | 세로 청록색 `PlaceholderSprite` | 캐릭터의 갑옷·무기와 재질 표현이 연결되지 않음 |
| 균열 | 붉은 반투명 세로 영역 | 적이 등장하는 세계 오브젝트로 인식되지 않음 |
| 접지 | 일부 캐릭터에 작은 선 또는 그림자 없음 | 캐릭터가 바닥에서 떠 보임 |
| 화면 구성 | 배경을 viewport 크기에 맞춰 X/Y 별도 확대 | 원본 픽셀과 형태가 왜곡되고 화면비마다 밀도가 달라짐 |

직접 원인은 `LaneManager.EnsureRuntimeBattlefieldVisuals()`가 전투 논리와 임시 렌더링을 함께 담당하는 구조다. 이 함수는 배경을 목표 크기에 맞춰 X/Y로 별도 스케일하고, 레인·영역·슬롯을 `PlaceholderSprite`로 생성한다.

## 3. 아트 기준

### 3.1 묘사 우선순위

전장 안의 대비와 디테일은 다음 순서를 지킨다.

1. 영웅·보스
2. 일반 몬스터·수정체·균열
3. 스킬 이펙트·피격 피드백
4. 레인 바닥과 근경 소품
5. 원경 숲과 안개

배경은 캐릭터보다 단순해야 하지만 다른 화풍처럼 단순해서는 안 된다. 동일한 외곽선 처리와 명암 방향을 유지하면서 채도와 대비만 낮춘다.

### 3.2 공통 조명

- 기본 광원: 좌측 상단의 차가운 달빛
- 수정체 보조광: 좌측의 청색 룬광
- 균열 보조광: 우측의 보라색·자홍색 역광
- 캐릭터 접지부: 짙은 청록색 반투명 그림자
- 배경 최고 명도는 캐릭터 중간 명도보다 낮게 유지한다.

왼쪽에서 오른쪽으로 `안전한 청색 봉문 영역 → 중립 전투 영역 → 위험한 보라색 균열 영역`이 자연스럽게 읽혀야 한다.

### 3.3 전장 팔레트

| 용도 | 색상 | 적용 기준 |
| --- | --- | --- |
| 원경 암부 | `#071012` | 순수 검정 대신 숲과 안개에 사용 |
| 숲 중간톤 | `#13241F` | 나무, 덤불, 원경 폐허 |
| 이끼·바닥 | `#294238` | 레인 가장자리와 전경 소품 |
| 흙길 | `#40372D` | 세 레인의 기본 재질 |
| 석재 | `#39454A` | 수정체 제단과 균열 석문 |
| 황동 | `#B99048` | 제단 테두리와 봉문 장치에 제한적으로 사용 |
| 수정체 룬광 | `#45B7D9` | 수정체, 보호막, 아군 방향광 |
| 균열 암부 | `#261833` | 균열 내부와 오염 그림자 |
| 균열 발광 | `#8D5BE1` | 균열 핵과 적 등장 맥동 |
| 위험 강조 | `#D65C72` | 보스 웨이브와 균열 과부하에만 사용 |

### 3.4 텍스처 규칙

- 캐릭터와 동일하게 `Filter Mode: Point`, `Pixels Per Unit: 100`을 사용한다.
- Stage 1 신규 전장 자산은 Android에서도 `Compression: None` 또는 검증된 `Low Quality`만 허용한다.
- 외곽선은 완전한 검정 대신 짙은 청록·남색 계열을 사용한다.
- 한 오브젝트 안에서 최소 4단계 명암을 사용해 캐릭터와 재질 밀도를 맞춘다.
- 배경 전체에 단색 tint를 덮어 스타일을 맞추지 않는다. 원본 팔레트에서 직접 조정한다.
- 월드 오브젝트는 X/Y 비균등 스케일을 금지한다.

## 4. 전장 구성

```text
┌──────────────── BattlefieldViewport ────────────────┐
│ 원경: 봉문 숲 실루엣 · 안개 · 파손된 석조 구조물   │
│                                                     │
│ [청색 수정체 제단]  ===== 상단 흙길 =====  [균열문] │
│                     ===== 중단 흙길 =====            │
│                     ===== 하단 흙길 =====            │
│                                                     │
│ 근경: 이끼 낀 돌 · 뿌리 · 낮은 안개 · 화면 경계 암부│
└─────────────────────────────────────────────────────┘
```

### 4.1 원경

- 중앙 전투 영역은 캐릭터 실루엣을 방해하지 않도록 낮은 대비로 비운다.
- 나무와 폐허는 화면 상단과 좌우 경계에 집중한다.
- 세로 화면에서 보이는 위·아래 여유 공간에는 안개, 수풀, 파손된 봉문석을 배치해 검은 공백을 없앤다.
- 가로 fallback에서는 좌우가 더 보이므로 수정체 제단과 균열문의 외곽 구조까지 자연스럽게 이어져야 한다.

### 4.2 레인

- 회색 직선과 단색 바를 제거한다.
- 하나의 타일 가능한 흙길·석판 스프라이트를 세 레인에 반복 사용한다.
- 레인 중심은 주변보다 8~12% 밝게 하되 명시적인 중앙선을 그리지 않는다.
- 이끼, 깨진 석판, 작은 청색 룬 조각으로 전투 경로를 암시한다.
- 영웅 배치 슬롯은 평상시 숨기고 배치·튜토리얼이 요구할 때만 바닥 룬 데칼을 표시한다.

### 4.3 수정체 제단

- 단순한 청록색 기둥이 아니라 `석재 받침 + 황동 링 + 청색 결정 + 룬 각인` 구조로 만든다.
- 높이는 세 레인 전체를 가리지 않고 중단 레인을 중심으로 위·아래 레인까지 존재감이 이어지는 크기로 제한한다.
- 피격 판정 위치는 유지하고 표현 루트만 별도로 둔다.
- 상태 표현:
  - 기본: 약한 청색 내부광
  - 보호막: 외곽 룬 링과 얇은 원형 실드
  - 피격: 기존 흰색 플래시 + 짧은 청색 파편
  - 위험 HP: 결정 내부광이 청색에서 적색으로 점멸

### 4.4 균열문

- 붉은 영역 사각형 대신 `부서진 석문 + 보랏빛 공간 균열`을 배치한다.
- 석문은 수정체 제단과 같은 석재·황동 문법을 사용하되 파손되고 오염된 상태로 만든다.
- 균열 중심은 몬스터보다 밝지 않게 유지하고 가장자리만 발광시킨다.
- 상태 표현:
  - 대기: 1.8초 주기의 약한 맥동
  - 웨이브 예고: 0.4초 동안 밝기와 크기 8% 증가
  - 보스 웨이브: 자홍색 균열과 작은 파편 효과 추가

### 4.5 캐릭터 접지

- 모든 영웅·몬스터 아래에 동일 계열의 타원형 접지 그림자를 적용한다.
- 그림자 폭은 캐릭터 시각 폭의 45~60%, 높이는 폭의 18~24%로 한다.
- 알파는 영웅 0.42, 일반 몬스터 0.36, 보스 0.48을 기준으로 한다.
- 비행형 몬스터는 그림자를 더 흐리게 하고 본체와 그림자 사이 거리를 늘린다.

## 6. Unity 적용 구조

### 6.1 책임 분리

`LaneManager`는 좌표와 전투 범위만 계산하고 전장 그래픽을 생성하지 않는다.

새 구성:

```text
BattlefieldArtRoot
├─ BackdropLayer       sorting -30
├─ FarAtmosphereLayer  sorting -25
├─ GroundLayer         sorting -20
│  ├─ LaneTop
│  ├─ LaneMiddle
│  └─ LaneBottom
├─ GroundDecalLayer    sorting -10
├─ ObjectiveLayer      sorting 2~3
│  ├─ SealCrystalVisual
│  └─ SpawnRiftVisual
├─ UnitLayer           sorting 4~8, 기존 캐릭터
└─ WorldEffectLayer    sorting 10~15
```

### 6.2 새 데이터와 컴포넌트

#### `BattlefieldArtTheme` ScriptableObject

- 배경, 레인, 수정체, 균열, 그림자, 배치 룬 스프라이트
- 전장 팔레트
- 정렬 순서
- 수정체·균열 모션 시간과 발광 강도
- 세로·가로 표현 스케일

#### `BattlefieldVisualController`

- `LaneManager`의 좌표를 받아 시각 요소를 한 번 생성한다.
- 배경은 비율을 유지한 `cover` 방식으로 맞추고 남는 영역을 크롭한다.
- 레인은 타일 또는 반복 배치하고 비균등 확대하지 않는다.
- 수정체 판정 Transform과 시각 Transform을 분리한다.
- 화면비 변경 시 배경 크롭과 레인 길이만 다시 계산한다.
- 균열 맥동은 `Time.unscaledTime`을 사용한다.

공개 인터페이스:

- `BattlefieldVisualController.IsReady`
- `BattlefieldVisualController.Configure(BattlefieldArtTheme, LaneManager, CrystalController)`
- `BattlefieldVisualController.RefreshLayout()`
- `BattlefieldVisualController.SetRiftState(BattlefieldRiftState)`
- `BattlefieldVisualController.SetCrystalState(BattlefieldCrystalState)`
- `BattlefieldVisualController.CreateUnitShadow(Transform, SpriteRenderer, UnitVisualKind)`

#### 상태 열거형

- `BattlefieldRiftState`: `Idle`, `WaveWarning`, `BossWarning`, `Dormant`
- `BattlefieldCrystalState`: `Normal`, `Shielded`, `Damaged`, `Critical`, `Destroyed`
- `UnitVisualKind`: `Hero`, `Monster`, `FlyingMonster`, `Boss`

### 6.3 기존 코드 변경 범위

| 파일 | 변경 방향 |
| --- | --- |
| `LaneManager.cs` | `EnsureRuntimeBattlefieldVisuals()`의 렌더링 생성 책임 제거 |
| `RuneGateBootstrapper.cs` | 전장 테마·프리팹 생성 및 BattleScene 연결 |
| `CrystalController.cs` | 상태 이벤트는 유지하고 시각 상태 이벤트만 추가 |
| `HeroPlacementManager.cs` | 영웅 생성 시 공통 접지 그림자 요청 |
| `WaveManager.cs` | 몬스터 생성 시 그림자 요청, 웨이브 예고 시 균열 상태 전달 |
| `BattlefieldCameraFitter.cs` | viewport 계산 후 `RefreshLayout()` 호출 |
| `RuneGateProjectValidator.cs` | 신규 자산·테마·프리팹·비균등 스케일 금지 검사 |
| `RuneGateVisualCaptureRunner.cs` | 기본·균열 예고·보호막·보스 예고 캡처 추가 |

전투 로직은 `BattlefieldVisualController`를 몰라도 동작해야 한다. 단, 새 BattleScene에서는 테마나 컨트롤러가 누락되면 임시 사각형으로 돌아가지 않고 명확한 오류를 출력한다.

## 7. 화면비 대응

### 720×1280 / 1080×1920

- 중앙 3개 레인과 수정체·균열이 모두 보이도록 배경 좌우를 크롭한다.
- 원경 상단은 최소 12%, 근경 하단은 최소 8%를 유지한다.

### 1080×2400

- 추가 세로 공간은 검은 공백이 아니라 숲 안개와 전경 뿌리로 채운다.
- 캐릭터와 레인의 실제 월드 좌표는 변경하지 않는다.

### 1600×900 fallback

- 배경 상하를 크롭하고 좌우의 수정체 제단·균열문 외곽 구조를 더 노출한다.
- 세 레인의 간격과 캐릭터 표현 높이는 기존 fallback 값을 유지한다.

## 8. 적용 순서

1. 현재 캐릭터 6명과 몬스터 3종을 기준으로 팔레트·외곽선·명암 레퍼런스 시트를 확정한다.
2. 배경, 레인, 수정체, 균열의 정지 이미지 시안을 한 장의 전장 합성본으로 검토한다.
3. 최소 자산 8종을 제작하고 import preset을 통일한다.
4. `BattlefieldArtTheme`과 `BattlefieldVisualController`를 구현한다.
5. `LaneManager`의 임시 사각형 렌더링을 제거한다.
6. 수정체·균열 상태와 유닛 접지 그림자를 연결한다.
7. 네 해상도 캡처와 Android System/Full Chapter E2E를 재검증한다.

2단계 합성본이 승인되기 전에는 Unity 코드 통합을 시작하지 않는다. 배경 시안이 잘못된 상태에서 레이아웃 코드까지 맞추면 재작업 범위가 커지기 때문이다.

## 9. 완료 조건

- 캐릭터, 몬스터, 수정체, 균열이 동일한 명암 방향과 외곽선 밀도로 보인다.
- 레인이 회색 바나 디버그 선처럼 보이지 않는다.
- 수정체와 균열에 반투명 사각형 영역이 노출되지 않는다.
- 모든 유닛이 바닥에 접지되어 보인다.
