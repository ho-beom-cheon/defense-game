# Battle uGUI 아트 디렉션 v1

## 목표

Stage 1 전투 화면을 `봉문 전술 기록서`로 정의한다. 플레이어가 첫 화면에서 스테이지, 수정체 상태, 전장, 여섯 영웅의 스킬 순서를 즉시 읽을 수 있어야 한다. 장식보다 전투 정보의 계층과 터치 안정성을 우선한다.

## 디자인 토큰

| 용도 | 값 |
|---|---|
| 배경 | `#0B1117` |
| 기본 패널 | `#111C22` |
| 보조 패널 | `#18262D` |
| 황동 프레임·주요 CTA | `#B99048` |
| 룬·수정체 | `#45B7D9` |
| 성공 | `#4CCF7A` |
| 위험 | `#D65C5C` |
| 본문 | `#F2F0E8` |
| 보조 글자 | `#A9B7BC` |

- 글꼴은 Noto Sans KR Regular, Semibold, Bold를 TMP 동적 폰트로 사용한다.
- 간격 단위는 `8/16/24/32/48`, 최소 터치 높이는 기준 캔버스에서 `88`이다.
- 패널, 버튼, 룬 카드의 9-slice 테두리는 각각 `24/12/24px`이다.
- 황동색은 큰 프레임과 주요 행동에만 쓰고, 반복 카드의 주 강조색은 청색·상태색으로 제한한다.

## 화면 구조

```text
BattleCanvas · 1080×1920, Match 0.5
└─ SafeAreaRoot
   └─ BattleRoot · 최대 1080×2160
      ├─ HudLayer · Stage / Crystal / Wave+Gold / Pause
      ├─ BattlefieldViewport · 실제 카메라 viewport
      ├─ SkillLayer · 3×2 영웅 스킬 카드
      └─ OverlayLayer · Wave / Tutorial / Rune / Pause / Result
```

- 세로 화면은 HUD 152, 간격 16, 스킬 영역 400을 기준으로 하고 나머지를 전장으로 사용한다.
- 낮은 화면과 가로 fallback에서는 비례 축소하되 스킬 카드 터치 높이 88을 유지한다.
- 전투 카메라는 IMGUI 계산값이 아니라 `BattlefieldViewport`의 실제 화면 Rect를 사용한다.
- 모달 우선순위는 `Result > RuneSelection > Tutorial > Pause > None`이다.

## 상태별 기준

### 기본 전투

- 상단 HUD는 한 줄에서 Stage, 수정체 HP, 웨이브·전리품, 일시정지를 읽게 한다.
- 전장은 가장 큰 면적을 차지하고 수정체와 오른쪽 균열이 프레임 안에 완전히 보여야 한다.
- 하단 3×2 스킬 카드는 영웅 초상, 영웅명, 스킬명, 상태, HP, 쿨다운을 한 카드 안에서 읽게 한다.

### 튜토리얼

- 배경 입력을 차단하고 기록 번호, 제목, 본문, 이전·다음·건너뛰기 순으로 읽힌다.
- 캡처 기준은 1단계 수정체, 4단계 스킬, 7단계 업그레이드 안내다.

### 룬 선택

- 안전 너비 720 이상은 3열, 미만은 1열이다.
- 카드의 시선 순서는 속성 인장, 룬명, 희귀도·속성, 설명이다.
- 선택 중 다른 전투 입력은 차단한다.

### 결과

- 승패 제목과 획득 골드가 먼저 보이고, 전투 통계와 해금·조각 보상은 그 아래에 배치한다.
- 승리 시 다음 스테이지, 마지막 스테이지 또는 패배 시 재시도를 주요 행동으로 사용한다.

## QA 캡처 기준

- Windows: `720×1280`, `1080×1920`, `1080×2400`, `1600×900`.
- Android: API 35, `1080×2400`.
- 자동 캡처 라벨: `03-battle`, `04a-tutorial-1`, `04b-tutorial-4`, `04c-tutorial-7`, `04d-rune-selection`, `05-defeat-result`, `05b-victory-result`.
- 캡처 검토 시 글자 잘림, 버튼 겹침, 수정체·균열 잘림, 여섯 스킬 카드 노출, 모달 배경 입력 차단을 확인한다.
