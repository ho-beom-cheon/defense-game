The goal is to connect the game flow outside battle:

Title
→ Stage Select
→ Battle
→ Result
→ Upgrade
→ Save
→ Stage Select

This version should make the prototype feel like an actual game shell, not just a standalone battle scene.

Scope:

1. Scene flow

Create or update these scenes:
- Assets/_Project/Scenes/TitleScene.unity
- Assets/_Project/Scenes/StageSelectScene.unity
- Assets/_Project/Scenes/BattleScene.unity
- Assets/_Project/Scenes/UpgradeScene.unity

If scene generation is risky in code, create Editor bootstrap tools that generate them safely.

Scene requirements:

TitleScene:
- Shows game title: RuneGate Defense
- Has Start button
- Has Continue button if save data exists
- Has Reset Save button for development testing
- Start/Continue should move to StageSelectScene

StageSelectScene:
- Shows available stages
- At minimum, show Stage 1, Stage 2, Stage 3 placeholders
- Stage 1 should be playable by default
- Stage 2 unlocks after Stage 1 clear
- Stage 3 unlocks after Stage 2 clear
- Locked stages should be visible but not selectable
- Selecting a stage should start BattleScene with the selected StageData

BattleScene:
- Should no longer rely only on hardcoded sample stage loading if avoidable
- It should accept selected StageData from a central session/context object
- If no selected stage exists, fall back to Goblin Forest 1 sample stage
- On Victory or Defeat, go to Result flow instead of only showing a local placeholder panel
- Keep the existing battle behavior working

Result flow:
- Show Victory or Defeat
- Show earned gold
- Show stage clear status
- Buttons:
  - Retry
  - Upgrade
  - Stage Select
- Victory should mark the selected stage as cleared
- Victory should unlock the next stage
- Gold should be saved

UpgradeScene:
- Show player total gold
- Show upgrade list
- Allow buying upgrades if enough gold
- Save purchased upgrade levels
- Return to Stage Select

2. Save system

Implement local save only.

Create:
- SaveData.cs
- SaveManager.cs
- PlayerProgress.cs or equivalent if useful

Save location:
- Use Application.persistentDataPath
- Store JSON save file locally
- Do not use cloud save
- Do not use server
- Do not use PlayerPrefs for the main save if a JSON save manager exists

SaveData should include:
- int totalGold
- List<string> clearedStageIds
- List<string> unlockedStageIds
- Dictionary<string, int> upgradeLevels or serializable equivalent
- string lastSelectedStageId
- bool hasSeenIntro optional

Because Unity JsonUtility does not serialize Dictionary directly, use a serializable list structure if needed.

Required SaveManager behavior:
- Load save on startup
- Create default save if no save exists
- Save after:
  - stage clear
  - gold change
  - upgrade purchase
  - reset save
- Reset save for development testing
- Add null guards and warnings
