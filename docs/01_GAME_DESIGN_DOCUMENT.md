# RuneGate Defense 게임 디자인 문서

## 1. 게임 콘셉트

`RuneGate Defense`는 판타지 왕국을 배경으로 한 세로형 2D 로그라이트 디펜스 게임이다. 플레이어는 왕국 방어대장이 되어 룬게이트에서 쏟아져 나오는 몬스터를 막는다. 영웅은 자동으로 공격하고, 플레이어는 스킬 사용과 웨이브 사이 룬 선택으로 전투 방향을 결정한다.

## 2. 핵심 플레이 요약

```text
스테이지 선택
→ 전투 시작
→ 영웅 자동 공격
→ 몬스터 웨이브 방어
→ 웨이브 종료
→ 룬 카드 3개 중 1개 선택
→ 다음 웨이브
→ 보스 웨이브
→ 승리/패배
→ 보상 및 영구 성장
→ 재도전
```

## 3. 플레이 시간

| 구분 | 목표 |
|---|---:|
| 1스테이지 플레이 시간 | 2~3분 |
| 1웨이브 시간 | 20~40초 |
| 웨이브 수 | MVP 기준 2~5개 |
| 세션 길이 | 5~10분 내 반복 가능 |

## 4. 전투 화면 구조

## 5. 전투 핵심 규칙

### 5.1 승리 조건

- 모든 웨이브의 스폰이 끝나고, 살아있는 몬스터가 모두 제거되면 승리한다.
- 보스 웨이브가 있는 스테이지에서는 보스 처치가 승리 조건이다.

### 5.2 패배 조건

- 몬스터가 왕국 수정에 도달하면 수정 HP가 감소한다.
- 수정 HP가 0 이하가 되면 패배한다.

### 5.3 웨이브 규칙

- 웨이브는 `StageData`의 `WaveData` 리스트로 구성한다.
- 각 웨이브는 여러 개의 `WaveSpawnData`를 가진다.
- `WaveSpawnData`는 몬스터 종류, 라인, 수량, 시작 딜레이, 스폰 간격을 가진다.

### 5.4 웨이브 사이 룬 선택

- 마지막 웨이브가 아니면 웨이브 클리어 후 `RuneSelection` 상태로 전환한다.
- 룬 카드 3장이 표시된다.
- 유저가 1장을 선택하면 효과를 즉시 적용하고 다음 웨이브를 시작한다.

## 6. 핵심 재미 요소

### 6.1 매판 달라지는 빌드

룬 선택으로 같은 스테이지도 다른 전투 양상이 나오게 한다.

예시:

- 궁수 공격속도 빌드
- 기사 탱킹 빌드
- 마법 광역딜 빌드
- 수정 회복 안정 빌드
- 스킬 쿨타임 감소 빌드

### 6.2 짧은 판과 반복 성장

실패해도 골드를 얻고 영구 강화가 가능해야 한다. 이 구조가 재도전 동기를 만든다.

### 6.3 캐릭터 수집 감각은 주되 가챠는 배제

영웅은 확정 해금 또는 콘텐츠팩 구매로 제공한다.

## 7. 세계관

왕국의 변방에 오래전 봉인된 차원문 `룬게이트`가 다시 열렸다. 게이트 너머의 몬스터들이 왕국의 수정을 노리고 침공한다. 왕국 수정은 게이트를 닫을 수 있는 유일한 에너지원이며, 플레이어는 각 종족의 영웅들을 지휘해 수정을 지켜야 한다.

## 8. 월드 구성

### 8.1 MVP 월드: 고블린 숲

| 요소 | 내용 |
|---|---|
| 테마 | 숲, 부족, 초반 몬스터 |
| 주요 적 | 고블린, 오크 |
| 보스 | 오크 워로드 |
| 핵심 학습 | 라인, 자동 공격, 룬 선택, Crystal HP |

### 8.2 확장 월드 후보

| 월드 | 테마 | 몬스터 | 보스 |
|---|---|---|---|
| 언데드 묘지 | 어둠, 부활 | 해골, 좀비, 망령 | 리치 |
| 화산 지대 | 화염, 지속 피해 | 불정령, 용암괴물 | 화룡 |
| 얼음 협곡 | 둔화, 빙결 | 설인, 얼음마법사 | 서리 거인 |
| 마계 균열 | 고난도 | 악마, 마족 기사 | 균열 군주 |
| 정령 숲 | 자연, 회복 | 나무괴물, 정령 | 고대수 |

## 9. 영웅 설계

### 9.1 MVP 영웅

| 영웅 | 역할 | 위치 | 전투 목적 |
|---|---|---|---|
| Knight | Tank | Front | 적을 막고 생존 |
| Archer | RangedDps | Back | 안정적인 단일 원거리 딜 |

### 9.2 1차 확장 영웅

| 영웅 | 역할 | 특징 |
|---|---|---|
| Mage | Mage | 광역 피해 |
| Priest | Healer | 수정 또는 영웅 회복 |
| Engineer | Engineer | 포탑/장벽 설치 |
| Assassin | Assassin | 보스 또는 고HP 적 우선 처치 |

### 9.3 장기 확장 방식

영웅은 종족, 직업, 속성 조합으로 확장한다.

```text
인간 + 기사 + 빛 = 성기사
엘프 + 궁수 + 바람 = 바람궁수
드워프 + 기술자 + 화염 = 화염포병
언데드 + 마법사 + 어둠 = 흑마법사
정령 + 성직자 + 빛 = 빛의 정령사
```

## 11. 룬 시스템

룬은 전투 중 임시 강화다. 해당 스테이지가 끝나면 사라진다.

### 11.1 MVP 룬

| 룬 | effectKey | 효과 |
|---|---|---|
| Sword Rune | `hero_attack_percent` | 모든 영웅 공격력 증가 |
| Bow Rune | `hero_attack_speed_percent` | 모든 영웅 공격속도 증가 |
| Healing Rune | `crystal_heal_flat` | 수정 HP 회복 |

### 11.2 확장 룬 후보

| 룬 | 효과 |
|---|---|
| Flame Rune | 공격 시 화상 피해 |
| Frost Rune | 적 이동속도 감소 |
| Lightning Rune | 주기적으로 랜덤 적에게 피해 |
| Guardian Rune | 수정 보호막 획득 |
| Mana Rune | 스킬 쿨타임 감소 |
| Hunter Rune | 보스 피해 증가 |

## 12. 스킬 설계

### Knight: Shield Bash

- 가장 가까운 적에게 피해
- 가능하면 짧은 기절 또는 넉백
- MVP에서는 피해만 구현해도 된다.

### Archer: Rapid Shot

- 가장 가까운 적 또는 가장 HP 높은 적에게 연속 피해
- MVP에서는 즉시 3회 피해로 처리해도 된다.

## 13. 영구 성장 설계

MVP 이후 도입한다.

| 성장 | 효과 |
|---|---|
| Crystal 강화 | 수정 최대 HP 증가 |
| 영웅 공격력 강화 | 모든 영웅 공격력 증가 |
| 영웅 체력 강화 | 모든 영웅 HP 증가 |
| 시작 골드 증가 | 전투 시작 보너스 증가 |
| 스킬 쿨타임 감소 | 모든 스킬 쿨타임 감소 |
| 룬 품질 증가 | 희귀 룬 등장 확률 증가 |

## 14. 수익화 장기 방향

MVP에는 수익화가 없다. 출시 이후 고려한다.

- 광고 제거
- 확정형 영웅팩
- 확정형 월드팩
- 스킨팩
- 추가 스테이지팩

하지 않는다.

- 확률형 영웅 뽑기
- 확률형 장비 뽑기
- 서버 기반 경쟁 랭킹 BM
