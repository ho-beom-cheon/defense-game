# RuneGate Defense v0.7 Core Content Balance

v0.7의 목표는 Stage 1~10을 실제로 진행 가능한 첫 콘텐츠 곡선으로 정리하는 것이다. 전투 시스템은 기존 BattleScene 흐름을 유지하고, 데이터와 룬 효과만 확장한다.

## 적용 범위

- 영웅 6종의 전투 스탯과 스킬 수치를 v0.7 기준으로 정리했다.
- 몬스터 6종과 보스 1종의 HP, 이동 속도, 크리스탈 피해량, 골드 보상을 정리했다.
- 룬 20장을 요청된 이름과 effectKey 기준으로 재구성했다.
- Stage 1~10은 초반 성장감과 후반 조합 압박을 기준으로 유지/조정했다.
- 업그레이드 비용은 Lv.1 80, Lv.2 150, Lv.3 280, Lv.4 480, Lv.5 800에 가까운 곡선이 되도록 `baseCost 80`, `costMultiplier 1.8`, `maxLevel 5`로 정리했다.

## 실제 구현된 룬 effectKey

아래 10개 이상은 BattleScene 런타임에서 실제 효과 또는 안전한 단순화 효과를 가진다.

- `hero_attack_percent`
- `hero_attack_speed_percent`
- `crystal_heal_flat`
- `skill_cooldown_percent`
- `hero_max_hp_percent`
- `monster_slow_percent`
- `boss_damage_percent`
- `mage_area_percent`
- `tank_defense_percent`
- `turret_attack_percent`

일부 룬은 v1.0 이후 전용 시스템을 위한 placeholder hook으로 유지한다.

## 진행 곡선

- Stage 1~3은 빠른 클리어와 첫 업그레이드 구매를 우선한다.
- Stage 4~6은 빠른 적, 비행형, 분열형을 소개해 역할 가치를 보여준다.
- Stage 7~9는 3라인 압박과 혼합 웨이브로 룬 선택과 조합 중요도를 높인다.
- Stage 10은 그룸바르가 등장하는 첫 보스전이며, 클리어 가능성을 우선한다.

## 남은 밸런스 과제

- 실제 플레이타임 90~120초 기준은 Unity 수동 플레이로 추가 측정이 필요하다.
- 분열, 부활, 보스 소환, 보스 phase change는 현재 hook 또는 단순화 상태다.
- 룬 선택 확률과 rarity 체감은 플레이테스트 후 조정한다.
