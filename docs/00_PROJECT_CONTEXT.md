# RuneGate Defense 프로젝트 컨텍스트

> 이 문서는 Codex가 대화 맥락 없이도 프로젝트 방향을 이해하도록 하기 위한 최상위 컨텍스트 문서다. Codex는 구현 전에 이 문서를 먼저 읽고, 이후 세부 문서를 순서대로 읽어야 한다.

## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 프로젝트명 | RuneGate Defense |
| 장르 | 세로형 2D 로그라이트 판타지 디펜스 |
| 플랫폼 | Android 우선, 이후 iOS 검토 |
| 엔진 | Unity 6 |
| 언어 | C# |
| 수익화 방향 | 광고 제거, 월드팩, 영웅팩, 스킨팩. 단, MVP에는 수익화 기능 제외 |
| 서버 | MVP 단계에서는 없음 |
| 데이터 저장 | 로컬 저장 |
| 핵심 UX | 1판 2~3분, 자동 전투 + 수동 스킬 + 웨이브 후 룬 선택 |

## 2. 현재 구현 상태

현재 프로토타입은 다음까지 구현된 상태로 간주한다.

- `BattleScene`이 Unity에서 실행됨
- 3개 라인이 화면에 표시됨
- Crystal HP가 표시됨
- 현재 Wave, Battle State, Gold가 표시됨
- Knight / Archer 스탯이 HUD에 표시됨
- Knight / Archer 스킬 버튼이 표시됨
- Placeholder 형태의 영웅과 몬스터가 화면에 나타남
- 몬스터가 라인 위에서 이동함
- 초기 전투 흐름이 작동함

즉, 현재 단계는 **Battle Prototype v0.1**이다.

## 3. 최종 게임 핵심 문장

> 차원문 룬게이트에서 몰려오는 몬스터를 막기 위해 영웅을 배치하고, 웨이브마다 룬 카드를 선택해 강화하며, 왕국 수정을 지키는 세로형 2D 로그라이트 판타지 디펜스 게임.

## 4. 개발 철학

이 프로젝트는 개인 개발자가 장기 유지보수 가능한 구조로 만드는 것이 목표다. 따라서 다음 원칙을 따른다.

1. **작게 완성하고 확장한다.**
   - 처음부터 수십 명의 영웅, 대형 월드, 복잡한 강화 시스템을 만들지 않는다.
   - 먼저 1개 월드, 2명 영웅, 2종 몬스터로 게임 루프를 완성한다.

2. **서버 없이 시작한다.**
   - MVP에는 로그인, 계정, 랭킹, 클라우드 저장이 없다.
   - 모든 저장은 로컬 JSON 또는 PlayerPrefs 기반으로 처리한다.

3. **가챠를 넣지 않는다.**
   - 확률형 아이템, 확률형 영웅 뽑기는 MVP와 초기 출시 버전에서 제외한다.
   - 유료 판매는 확정형 콘텐츠팩으로 간다.

4. **일러스트는 마지막에 몰아넣지 않는다.**
   - 전투 로직은 placeholder로 먼저 완성한다.
   - 아트 규격과 폴더 구조는 초반부터 잡는다.
   - 대표 영웅/몬스터 1~2개를 중간에 먼저 적용해 파이프라인을 검증한다.

5. **밸런스 수치를 코드에 박지 않는다.**
   - 영웅, 몬스터, 룬, 스킬, 스테이지 수치는 `ScriptableObject`를 기본으로 한다.
   - 코드에는 동작 로직만 둔다.

6. **Codex가 이해 가능한 문서 기반 개발을 한다.**
   - Codex는 이전 대화 맥락이 없을 수 있다.
   - 따라서 모든 큰 방향, 제한사항, 구현 우선순위는 `docs/` 아래 md 파일에 기록한다.

## 5. MVP 제외 사항

MVP에는 절대 넣지 않는다.

- 서버
- 로그인
- 계정 시스템
- 클라우드 저장
- 랭킹
- 멀티플레이
- 실시간 PvP
- 가챠
- 확률형 아이템
- 광고
- 인앱결제
- Firebase
- Analytics
- Addressables
- 외부 유료 API
- 복잡한 튜토리얼
- 고퀄리티 일러스트 대량 적용
- 드래그 앤 드롭 영웅 배치

## 6. MVP 포함 사항

MVP에는 반드시 포함한다.

- 3라인 전투
- 왕국 수정 HP
- 몬스터 웨이브 스폰
- 영웅 자동 공격
- 수동 스킬 버튼
- 웨이브 종료 후 룬 카드 3택1
- 룬 효과 적용
- 승리/패배 화면
- 재시작 버튼
- 로컬 저장 기반 영구 강화 기초
- Placeholder 기반 시각 피드백

## 7. 현재 가장 중요한 다음 목표

현재 다음 목표는 **Battle Prototype v0.2**다.

v0.2의 핵심은 다음이다.

1. 웨이브 종료 감지
2. 룬 선택 UI 표시
3. 룬 선택 후 효과 적용
4. 다음 웨이브 시작
5. 마지막 웨이브 클리어 시 Victory
6. Crystal HP 0 시 Defeat
7. Restart 가능
8. 몬스터 HP Bar 및 피격 피드백
9. 스킬 쿨타임 UI 개선

## 8. Codex 작업 기본 규칙

Codex는 다음 규칙을 따라야 한다.

- 구현 전 `docs/00_PROJECT_CONTEXT.md`를 읽는다.
- 관련 세부 문서를 읽고 작업 범위를 좁힌다.
- 작업 전에 짧은 구현 계획을 출력한다.
- 외부 패키지를 추가하지 않는다.
- Unity 컴파일 가능성을 우선한다.
- 에러가 발생하면 최소 변경으로 고친다.
- 작업 후 변경 파일 목록과 수동 테스트 방법을 보고한다.
- 테스트를 실제로 실행하지 못했다면 “실행하지 못했다”고 명확히 말한다.
