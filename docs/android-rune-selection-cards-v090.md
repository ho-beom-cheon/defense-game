# Android Rune Selection Cards v0.9.0

## 작업 범위

- 관련 이슈: `#77`
- 작업 브랜치: `codex/issue-77-rune-selection-cards`
- 웨이브 사이 룬 선택 화면을 세로 텍스트 목록에서 3장의 전술 기록 카드로 재구성했다.
- 기존 RuneManager 선택 처리, BattleState 전환, 다음 웨이브 시작 규칙은 변경하지 않았다.

## 카드 구성

- 한국어 룬 이름
- 일반, 희귀, 영웅 희귀도와 구분 색상
- 화, 빙, 뢰, 지, 광, 영, 독, 풍, 무 속성 문양과 색상
- 한국어 설명과 실제 효과 수치
- 카드 전체 폭의 선택 버튼
- 선택 요청 후 `적용 중` 및 중복 입력 잠금

`RuneData.Icon`이 있으면 카드 문양에 사용한다. 아이콘이 없으면 속성 색상 테두리와 한글 한 글자 문양으로 폴백한다. `effectKey`는 디버그 옵션을 켠 경우에만 표시한다.

## 정적 검증

`RuneGateProgressionSmokeTest`는 720x1280, 1080x1920, 1440x2560, 1600x900, 2048x1152에서 다음을 검사한다.

- 룬 팝업과 카드가 Safe Area 안에 위치하는지
- 카드 3장의 최소 크기와 중첩 여부
- 충분한 세로 화면 폭에서 3열을 사용하는지
- 일반, 희귀, 영웅 희귀도 색상이 구분되는지
- 모든 속성에 폴백 문양이 있는지

## 남은 한계

- 카드와 버튼은 기존 IMGUI 구조를 유지한다.
- 대부분의 룬은 전용 아이콘 대신 속성 문양 폴백을 사용한다.
- 카드 설명 영역은 짧은 문구에서 여백이 크며 최종 Canvas 전환 때 정보 밀도를 다시 조정할 수 있다.
- 실제 Android 기기의 화면 컷아웃, 시스템 글꼴 배율, 연속 터치 내구성은 추가 확인이 필요하다.
